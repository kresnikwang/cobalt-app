## 🔴 打包构建修复（多个问题串联排查）

### 根因链
1. **Vite workspace 包 external** → `@imput/version-info` 符号链接在 asar 中失效
2. **Electron CJS/ESM interop** → `import { app } from 'electron'` 在 Vite 打包后失败
3. **electron-builder 缺少 node_modules** → `includeSubNodeModules: true` 修复
4. **isolated-vm ABI 不匹配** → 系统 Node.js v22 (ABI 127) 编译的二进制与 Electron 31 (ABI 125) 冲突
5. **⚠️ isolated-vm 的 node-gyp-build 依赖不在 unpacked** → 从 unpacked 的 isolated-vm.js 中 `require('node-gyp-build')` 找不到 asar 中的模块

### 最终修复方案

6. **packages/desktop/vite.config.ts**
   - external 函数例外：`@imput/version-info` 标记为 bundled

7. **packages/desktop/package.json** — electron-builder 配置
   - 添加 `npmRebuild: false` — 禁止用系统 Node.js 重编译 native 模块
   - 添加 `includeSubNodeModules: true` — 包含所有子 node_modules
   - 添加 `afterPack` hook — 删除 packaged app 中 `build/Release/isolated_vm.node`
   - asarUnpack 添加 `**/node_modules/node-gyp-build/**/*` — isolated-vm 的依赖必须也 unpack

8. **packages/desktop/scripts/fix-isolated-vm.mjs** — 新增
   - 删除 `node_modules/isolated-vm/build/Release/` 让 node-gyp-build 使用 prebuilds
   - 搜索 root + pnpm store 中的 isolated-vm 安装
   - 作为 `postinstall` 和 `build` 前置步骤运行

9. **packages/desktop/scripts/electron-builder-after-pack.cjs** — 新增
   - afterPack hook：删除 packaged .app 中的 `build/Release/isolated_vm.node`

### 验证
- `isolated-vm@6.1.2` 的 `prebuilds/darwin-arm64/isolated-vm.abi125.node` 匹配 Electron 31 (Node ABI 125)
- 终端直接测试 `isolated-vm` 加载成功（`[ 'lib' ]`）
- `node-gyp-build` 已 unpack 到 `app.asar.unpacked/node_modules/`
