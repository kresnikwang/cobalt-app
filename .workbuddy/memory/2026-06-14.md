## 🔴 打包构建修复（多个问题串联排查）

### 根因链
1. **Vite workspace 包 external** → `@imput/version-info` 符号链接在 asar 中失效
2. **Electron CJS/ESM interop** → `import { app } from 'electron'` 在 Vite 打包后失败
3. **electron-builder 缺少 node_modules** → `includeSubNodeModules: true` 修复
4. **isolated-vm ABI 不匹配** → 系统 Node.js v22 (ABI 127) 编译的二进制与 Electron 31 (ABI 125) 冲突
5. **⚠️ isolated-vm 的 node-gyp-build 依赖不在 unpacked** → 从 unpacked 的 isolated-vm.js 中 `require('node-gyp-build')` 找不到 asar 中的模块

### 最终修复方案

6. **packages/desktop/vite.config.ts**
   - external 函数例外：`@imput/version-info` 标记为 bundled

7. **packages/desktop/package.json** — electron-builder 配置
   - 添加 `npmRebuild: false` — 禁止用系统 Node.js 重编译 native 模块
   - 添加 `includeSubNodeModules: true` — 包含所有子 node_modules
   - 添加 `afterPack` hook — 删除 packaged app 中 `build/Release/isolated_vm.node`
   - asarUnpack 添加 `**/node_modules/node-gyp-build/**/*` — isolated-vm 的依赖必须也 unpack

8. **packages/desktop/scripts/fix-isolated-vm.mjs** — 新增
   - 删除 `node_modules/isolated-vm/build/Release/` 让 node-gyp-build 使用 prebuilds
   - 搜索 root + pnpm store 中的 isolated-vm 安装
   - 作为 `postinstall` 和 `build` 前置步骤运行

9. **packages/desktop/scripts/electron-builder-after-pack.cjs** — 新增
   - afterPack hook：删除 packaged .app 中的 `build/Release/isolated_vm.node`

## 🔴 HTTP 代理支持修复（YouTube 等被墙服务无法连接）

### 根因
1. Cobalt API 已内置 `HTTP_PROXY`/`HTTPS_PROXY` 代理支持（undici `EnvHttpProxyAgent`），但 Electron 桌面端启动 API 时**从未设置代理环境变量**
2. `api/src/stream/proxy.js` 使用 `new Agent()` 而非全局 dispatcher → 绕过了 `EnvHttpProxyAgent` 代理
3. `api/src/processing/helpers/youtube-session.js` 使用 `new Agent()` → YouTube poToken session server 也绕过代理

### 修复方案
1. **packages/desktop/src/main/index.ts**
   - Settings 新增 `proxyEnabled: boolean` 和 `proxyUrl: string`
   - 默认值：`proxyEnabled: true`, `proxyUrl: 'http://127.0.0.1:7897'`
   - 在 `startCobaltServer()` 中根据 settings 设置 `HTTP_PROXY`/`HTTPS_PROXY`/`NO_PROXY`
   - `NO_PROXY=localhost,127.0.0.1,::1` 避免本地连接走代理
   - 新增 `restart-app` IPC handler（`app.relaunch() + app.exit(0)`）

2. **api/src/stream/proxy.js**
   - `import { Agent, request }` → `import { getGlobalDispatcher, request }`
   - `dispatcher: defaultAgent` → `dispatcher: getGlobalDispatcher()`

3. **api/src/processing/helpers/youtube-session.js**
   - `import { Agent }` → `import { getGlobalDispatcher }`
   - `dispatcher: defaultAgent` → `dispatcher: getGlobalDispatcher()`

4. **UI 代理设置**
   - App.svelte 新增代理开关和 URL 输入框
   - 设置面板添加 "Restart App" 按钮
   - 添加分隔线和区域标题样式

5. **i18n 中文翻译**
   - 新增 `zh.json` 完整中文翻译
   - en/ru 也添加代理相关 key

### 验证
- `curl -x http://127.0.0.1:7897 http://www.youtube.com/` 代理连通
- `HTTP_PROXY=http://127.0.0.1:7897 HTTPS_PROXY=http://127.0.0.1:7897 node api/src/cobalt.js` → YouTube 视频解析成功
- Rick Astley - Never Gonna Give You Up (4K Remaster) → tunnel URL 正常返回

## 🔴 EnvHttpProxyAgent 显式安装 + macOS 签名问题

### 发现：仅设置 HTTP_PROXY 环境变量不够
- undici.fetch() 只在全局 dispatcher 是 EnvHttpProxyAgent 时才走代理
- 设置 `process.env.HTTP_PROXY` 后还需显式 `new EnvHttpProxyAgent()` + `setGlobalDispatcher()`
- 验证：Electron Node 模式下 `fetch('https://www.youtube.com/')` → 200 OK（经 Clash Verge 代理）

### macOS 26.5 (Sequoia) 签名限制
- electron-builder 用 `identity: null` 跳过签名
- 但 macOS 26 会静默杀死未签名应用（无崩溃报告、无错误输出）
- `codesign --force --deep -s -` + `xattr -cr` 不够，macOS 仍会阻止
- 需要用户在系统偏好设置→隐私与安全性→手动允许，或用正确证书签名
