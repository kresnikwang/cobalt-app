## 🔴 HTTP 代理支持修复（YouTube 等被墙服务无法连接）

### 根因
1. Cobalt API 已内置 `HTTP_PROXY`/`HTTPS_PROXY` 代理支持（undici `EnvHttpProxyAgent`），但 Electron 桌面端启动 API 时**从未设置代理环境变量**
2. `api/src/stream/proxy.js` 使用 `new Agent()` 而非全局 dispatcher → 绕过了 `EnvHttpProxyAgent` 代理
3. `api/src/processing/helpers/youtube-session.js` 使用 `new Agent()` → YouTube poToken session server 也绕过代理

### 修复方案
1. **packages/desktop/src/main/index.ts**
   - Settings 新增 `proxyEnabled: boolean` 和 `proxyUrl: string`
   - 默认值：`proxyEnabled: true`, `proxyUrl: 'http://127.0.0.1:7897'`
   - 在 `startCobaltServer()` 中根据 settings 设置 `HTTP_PROXY`/`HTTPS_PROXY`/`NO_PROXY`
   - `NO_PROXY=localhost,127.0.0.1,::1` 避免本地连接走代理
   - 新增 `restart-app` IPC handler（`app.relaunch() + app.exit(0)`）

2. **api/src/stream/proxy.js**
   - `import { Agent, request }` → `import { getGlobalDispatcher, request }`
   - `dispatcher: defaultAgent` → `dispatcher: getGlobalDispatcher()`

3. **api/src/processing/helpers/youtube-session.js**
   - `import { Agent }` → `import { getGlobalDispatcher }`
   - `dispatcher: defaultAgent` → `dispatcher: getGlobalDispatcher()`

4. **UI 代理设置**
   - App.svelte 新增代理开关和 URL 输入框
   - 设置面板添加 "Restart App" 按钮
   - 添加分隔线和区域标题样式

5. **i18n 中文翻译**
   - 新增 `zh.json` 完整中文翻译
   - en/ru 也添加代理相关 key

### 验证
- `curl -x http://127.0.0.1:7897 http://www.youtube.com/` 代理连通
- `HTTP_PROXY=http://127.0.0.1:7897 HTTPS_PROXY=http://127.0.0.1:7897 node api/src/cobalt.js` → YouTube 视频解析成功
