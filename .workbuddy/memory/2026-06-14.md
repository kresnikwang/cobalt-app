# 2026-06-14

## YouTube 长视频下载修复 + 默认质量改 720p

### 修改文件
1. **api/src/stream/internal.js** — 3处改动
   - 分块大小：YouTube 专用 2MB → 统一 8MB（减少 75% 的 HTTP Range 请求数）
   - transplant 阈值：3 次 403 → 1 次（更快换链恢复）
   - 清理 verbose debug 日志（headers 泄露风险 + 生产噪音），仅保留关键错误

2. **api/src/stream/ffmpeg.js** — 1处改动
   - stderr: `'inherit'` → `'ignore'`（避免干扰 pipe 流）

3. **web/src/lib/settings/defaults.ts** — 1处改动
   - 默认下载质量: `"1080"` → `"720"`（用户要求同步到 720p）

### 分析要点
- YouTube 非 HLS 下载走 `handleChunkedStream`，720p h264 视频约 500MB → 8MB 分块 = ~63 次请求，大幅降低限流/Token 过期风险
- YouTube URL 不跟随重定向是设计意图（保留签名 URL），transplant 降阈作为兜底
- `debug-range.js` 和 `test_youtube_ua.js` 是调试脚本，保留未删
